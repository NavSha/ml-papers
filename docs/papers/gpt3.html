Language Models are Few-Shot Learners

Brown et al., OpenAI · NeurIPS 2020 · arXiv 2005.14165

At 175 billion parameters, you can teach it a new task by typing three examples into the prompt — no training run, no weights touched, nothing remembered when you leave.

the aha · figure 1.2, redrawn

The curve you've already lived

You have lived this chart: add two or three examples to a prompt and the output snaps into the format you wanted. Figure 1.2 shows what that experience actually is — an emergent property of scale that nobody programmed. As examples are added, small models stay flat; at 175B parameters, roughly 10× anything before it, the curve climbs steeply. The paper's name for the climb is in-context learning: a pure next-token predictor picking up a new task from examples sitting in the prompt — no training run, zero gradient updates.

what it changed

The cheapest quality lever you own

This is where the prompt became a programming interface — the direct ancestor of prompt engineering, few-shot prompting, and treating English as a runtime. Few-shot examples remain the cheapest quality lever you own: try a handful of good examples in the prompt before reaching for fine-tuning. In-context learning is also the mechanism that makes RAG work at all — retrieved text changes the model's behavior at inference time, with no training. And it has a line item: those examples are paid tokens on every single call, which is exactly the cost that prompt caching amortizes.

honest limits

Conditioning, not training

The misconception: "the model learns from my prompts — it remembers what I taught it." It doesn't. In-context learning updates zero weights; nothing persists after the request ends. Every call starts from the same frozen model — which is why your examples must be re-sent (or cached) every time.

The honest picture is a vastly experienced contractor who has already seen every kind of job: show two or three examples of the format you want and they instantly produce a fourth in the same style — but they retain nothing when they leave, and next visit you show the examples again. And the trick is rented from scale: the flat lines in Figure 1.2 are the same examples doing nothing for smaller models.

Build it yourself

build it: $5 · one evening

  1. read

    Read the paper — or just Figure 1.2: accuracy vs number of in-context examples, per model size. You have personally lived this curve every time examples improved your prompt.

    1–2 hrs · $0

  2. run

    Plot the in-context-learning curve yourself: pick one task, run it at 0/1/5/10 shots against a strong API model and a small local base model, plot accuracy vs shots — ICL working at scale and barely existing below it, in one chart of your own.

    one evening · <$5
